# 七鲜商品内容策划（主图采集 + 商详卖点提取 + 步骤1-3）

用于视频生成前的前置内容策划，产出可直接用于 AI 工具的视频分镜脚本。

## 执行约束

- 先检查是否已提供商品主图；若未提供，必须先执行“主图采集”再进入后续步骤
- 必须执行“商详页卖点提取”，且覆盖页面文本与图片中的文字卖点
- 必须基于用户输入生成四个产物：`商详卖点提取`、`切入点`、`创意结构`、`分镜脚本`
- 切入点、创意结构、分镜脚本都必须输出 `3套方案`（A/B/C），且彼此差异化
- 每套方案必须标注 `适配场景` 与 `适配平台`（如抖音/小红书/视频号）
- 四个产物必须分别保存为独立文件，写入同一个商品目录
- 文件内容必须与用户输入强关联，不得输出与商品无关的泛化文案
- 视频脚本总时长必须 `<=12秒`
- 分镜脚本前置镜头需以场景叙事为主，不以商品主图为核心构图
- 商品与品牌仅允许在最后一镜集中露出（前置镜头不出现品牌名、logo、完整商品主体）
- 允许在必要场合出现人物与轻互动（如拿取、对视、递交、品尝），但不得喧宾夺主
- 若用户输入信息缺失，先用“最小可执行假设”补齐，并在文件末尾标注“假设项”

## 输入参数规范

执行前先整理以下输入（缺失可补默认值）：

```
商品名称:
skuId:
商品品类:
商品主图目录（可选，若为空则自动采集）:
商详页URL（可选，若为空则通过商品名称+skuId定位）:
核心卖点:
辅助卖点:
目标用户:
用户需求:
期望风格（可选）:
目标时长（默认12秒）:
输出根目录（默认 ~/7freshVedioGenerate）:
```

### 目录与文件规则（必须执行）

1. 创建商品目录：
   - 先尝试：`商品目录 = 输出根目录/商品名称-skuId`
   - 若目录已存在，自动递增序号：`商品名称-skuId-2`、`商品名称-skuId-3`...
   - 当次任务统一使用变量：`最终目录名`、`最终商品目录`
   - 示例：`~/7freshVedioGenerate/芭乐樱樱茶-308443-2`
2. 主图文件命名（采集或用户提供后统一命名）：
   - `最终目录名-1.jpg` ~ `最终目录名-5.jpg`（通常5张）
3. 生成并保存 6 个文件：
   - `最终目录名-商详卖点提取.md`
   - `最终目录名-切入点.md`
   - `最终目录名-创意结构.md`
   - `最终目录名-分镜脚本-A.md`
   - `最终目录名-分镜脚本-B.md`
   - `最终目录名-分镜脚本-C.md`
4. 主图与六个文档文件均保存到同一商品目录下

### 目录创建算法（防覆盖，必须执行）

```bash
# 伪代码
base_name="商品名称-skuId"
candidate="$输出根目录/$base_name"

if [ ! -d "$candidate" ]; then
  最终目录名="$base_name"
else
  n=2
  while [ -d "$输出根目录/${base_name}-${n}" ]; do
    n=$((n+1))
  done
  最终目录名="${base_name}-${n}"
fi

最终商品目录="$输出根目录/$最终目录名"
mkdir -p "$最终商品目录"
```

## 第零步：获取商品主图（未提供时必须执行）

### 目标
从 `https://7fresh.m.jd.com` 获取指定商品主图并保存到商品目录，作为后续脚本参考素材。

### 触发条件

- 若用户已提供主图且通过质量检查：跳过采集，直接复用
- 若用户未提供主图，或提供主图不完整/不合格：执行采集流程

### 主图已提供时的跳过采集判定规则（必须执行）

满足以下条件才可跳过采集：

- 数量要求：至少 3 张，推荐 5 张
- 格式要求：JPG/JPEG（最终统一保存为 `.jpg`）
- 质量要求：单张图片建议 >100KB，且可正常打开
- 内容要求：至少覆盖 3 类画面（商品全貌 / 细节特写 / 场景或包装）

若任一条件不满足，执行以下策略：

- 数量不足 3 张：直接触发采集流程
- 数量为 3-4 张且质量合格：可复用现有图，但建议补采到 5 张
- 存在格式不统一：统一转存为 `.jpg` 并重命名
- 文件命名不规范：按规则自动重命名（见下方命名规则）
- 图片损坏或过小：剔除后重新采集替补

### 操作步骤

1. **打开七鲜移动端网站**
   - `https://7fresh.m.jd.com`
   - 若已拿到商详页，可直接访问商详链接（示例：`https://7fresh.m.jd.com/detail.html/?storeId=1030847505&skuId=556215&promotionId=&source=searchList`）
   - 打开后先确认页面为移动端样式（m站），再执行后续操作

2. **切换浏览器为移动端模式（必须先执行）**
   - 启用设备模拟/移动端视图（如 iPhone 12/13，或 390x844 视口）
   - 建议使用移动端 UA，避免被渲染为 PC 样式
   - 切换后刷新页面，确认出现移动端布局（顶部搜索栏、移动端楼层流）

3. **搜索目标商品**
   - 通过商品名称搜索，或直接打开商品详情页 URL

4. **进入商详页并确认信息**
   - 核对商品名称
   - 获取并确认 `skuId`（通常在 URL 中，如 `skuId=308443`）

5. **下载商品主图**
   - 主图优先来源：商详页前两个楼层（顶部轮播图 + 紧随其后的商品展示楼层）
   - 使用图片下载方式优先，其次使用高质量截图
   - 下载全部主图（通常5张；如页面仅展示3-4张则按实际保存）

6. **统一命名并保存**
   - 保存目录：`最终商品目录`
   - 文件命名：`最终目录名-1.jpg` ~ `最终目录名-5.jpg`

### 采集防跑偏规则（硬性执行）

- 全流程仅允许在 `7fresh.m.jd.com` 域名内操作
- 禁止跳转或切换到京东主站（如 `www.jd.com`、`search.jd.com`、`item.jd.com`）
- 若发生跳转，必须立即返回七鲜商详页继续执行，不得在京东主站继续搜索
- 在“已进入七鲜商详页”后，下一步必须是“提取前两个楼层主图并下载”，不得插入无关页面访问
- 禁止在 PC 样式下执行主图采集；若样式异常必须先切回移动端并刷新

### 主图提取优先级（按顺序执行）

1. 商详首屏轮播图（第1楼层）
2. 商详第2楼层商品展示图
3. 若前两楼层不足5张，再从后续楼层补齐

### 下载执行细则（避免漏下载）

- 每下载1张立即重命名为 `最终目录名-序号.jpg` 并保存到商品目录
- 每完成1张下载后，检查文件是否写入成功，再进行下一张
- 完成下载后，立即执行数量与格式校验（见下方质量检查命令）
- 未达到最低3张时视为失败，必须重试采集

### 主图自动重命名规则（复用用户已提供图片时）

- 按主图优先级排序后重命名：
  1) 商品全貌
  2) 原料/质地特写
  3) 卖点细节
  4) 场景图
  5) 包装或补充细节
- 重命名结果统一为：
  - `最终目录名-1.jpg`
  - `最终目录名-2.jpg`
  - `最终目录名-3.jpg`
  - `最终目录名-4.jpg`
  - `最终目录名-5.jpg`

### 质量检查

下载完成后执行：

```bash
# 检查图片数量和大小
ls -lh ~/7freshVedioGenerate/商品名-skuId/

# 验证图片格式
file ~/7freshVedioGenerate/商品名-skuId/*.jpg
```

检查要点：

- [ ] 主图数量正确（通常5张）
- [ ] 图片格式为 JPG
- [ ] 图片大小合理（>100KB）
- [ ] 图片可以正常打开
- [ ] 主图命名符合 `最终目录名-序号.jpg`
- [ ] 图片来源优先来自商详前两个楼层
- [ ] 采集过程中未跳转到京东主站域名
- [ ] 采集全程为移动端样式（非PC样式）

### 第零步保存要求（必须落盘）

- 主图文件必须保存到商品目录，与后续 4 个文档同目录
- 若采集失败，必须在回传结果中标记失败原因并停止后续步骤
- 若发生域名跑偏（跳到非 `7fresh.m.jd.com`），必须在回传中记录并说明已回退处理

## 第0.5步：根据商详页面提取商品卖点（必须执行）

### 目标
从七鲜商详页中抽取“可用于视频表达的商品卖点”，并明确区分来源于页面文本与来源于图片文字（OCR）的卖点。

### 执行时机

- 在完成第零步后立即执行
- 即使用户已提供核心卖点，也必须执行一次，用于补全与校验

### 提取范围（必须覆盖）

1. 商详页可复制文本：
   - 商品标题、短描述、规格参数、详情模块文案、活动说明
2. 商详页图片中的文字信息（必须识别）：
   - 轮播图文案、详情长图、卖点海报、参数图、产地与工艺图
3. 与卖点相关的关键事实：
   - 原料、产地、工艺、口感、营养、规格、保鲜方式、食用场景

### 图片卖点识别规则（硬性执行）

- 不能只看页面可选中文本，必须逐屏检查包含文字的图片区域
- 对每张含文字图片提取可读文字并做结构化归纳（等价于 OCR 识别）
- 对识别文本执行去噪：删除纯口号、非商品事实、夸张但不可验证措辞
- 若图片文字模糊，先尝试放大或高质量截图再识别，仍不清晰则标注“低可信度”
- 同一卖点若在文本与图片重复出现，合并为一条并标注双来源

### 卖点结构化归档规则

每条卖点都要标记：

```
卖点名称：
卖点类型（原料/产地/工艺/口感/营养/规格/场景/服务）：
卖点描述（事实化，不写空泛形容词）：
来源（页面文本/图片文字/OCR双来源）：
证据片段（原文短句）：
可信度（高/中/低）：
可用于镜头方向（特写/场景/互动/收尾）：
```

### 卖点分级规则（用于后续脚本）

- 核心卖点：1-2 条，必须可视化、可感知、可在 3 秒内讲清
- 辅助卖点：2-4 条，支撑核心卖点，不与核心重复
- 禁止纳入：纯促销话术、无事实依据口号、无法落地到镜头的抽象语

### 与用户输入卖点的合并规则

- 若用户提供卖点与商详提取一致：保留并提升可信度
- 若用户提供卖点商详未出现：保留但标注“用户输入，待验证”
- 若商详新增高价值卖点：补入辅助卖点池，并优先用于中段镜头

### 第0.5步保存要求（必须落盘）

- 文件名：`最终目录名-商详卖点提取.md`
- 文件内容结构：
  - 商详页定位信息（URL、skuId、抓取时间）
  - 文本通道卖点提取
  - 图片文字通道卖点提取（含识别说明）
  - 去重后的核心卖点/辅助卖点清单
  - 与用户输入差异对比
  - 假设项（如有）

### 质量检查

- [ ] 已覆盖页面文本与图片文字两个通道
- [ ] 至少提取 3 条有效卖点（推荐 4-6 条）
- [ ] 每条卖点均有来源与证据片段
- [ ] 已完成“核心卖点 1-2 条 + 辅助卖点 2-4 条”分级
- [ ] 模糊/不确定信息已标注可信度

## 第一步：根据卖点与用户需求生成视频切入点

### 目标
输入商品卖点和用户需求，产出“可情绪共鸣+可场景承接+可转化”的视频切入点。
最终产出 3 套切入点方案（A/B/C），并标注适配场景与平台。

### 操作步骤

1. **收集输入信息（必须完整）**
   - 商品名称、品类（果蔬/肉类/水产/预制菜）
   - 商品卖点（优先使用“第0.5步去重后的核心/辅助卖点”）
   - 目标用户（家庭主妇/上班族/健身人群）
   - 用户需求（新鲜、健康、便捷、高性价比）

2. **完成信息拆解**
   - 提炼 `1个核心表达`（一句话说清“这款产品为什么值得拍”）
   - 提炼 `1个核心卖点 + 2-4个辅助卖点`
   - 匹配 `2-4个高频场景`（如晨跑后、野餐时、加班间隙、聚会）
   - 明确 `情绪主轴`（如心动/清爽/热恋/治愈）

3. **输出视频切入点（必须3套）**
   - 方案A：情绪驱动型切入（示例：按心情解锁春日）
   - 方案B：场景问题型切入（示例：一天四个时刻，一杯刚好）
   - 方案C：对比反差型切入（示例：同样补水，这杯更轻负担）
   - 每套切入点都要写清：卖点如何映射到情绪/场景，并标注适配场景与平台

### 切入点输出格式

每套切入点统一输出：
```
方案编号（A/B/C）：
切入点名称：
方案定位（一句话）：
适配场景（2-4个）：
适配平台（1-2个）：
目标人群：
核心痛点：
核心表达（一句话）：
卖点-情绪映射：
  卖点1 -> 情绪1
  卖点2 -> 情绪2
  卖点3 -> 情绪3
场景映射：
  情绪1 -> 场景1
  情绪2 -> 场景2
建议开场话术（8-16字）：
收尾金句（15-25字，提问式优先）：
预期转化动作（评论互动/加购/下单）：
```

### 第一步保存要求（必须落盘）

- 文件名：`最终目录名-切入点.md`
- 文件内容结构：
  - 输入信息摘要（注明卖点来自商详提取/用户输入）
  - 切入点方案 A（必填）
  - 切入点方案 B（必填）
  - 切入点方案 C（必填）
  - 三套方案对比（人群/场景/平台/转化动作）
  - 推荐优先方案（必填，说明原因）
- 结尾补充：`假设项（如有）`

### 质量检查

- [ ] 切入点与卖点一一对应
- [ ] 切入点明确目标人群
- [ ] 切入点能在3秒内讲清价值
- [ ] 至少包含1条“卖点-情绪-场景”闭环
- [ ] 至少包含1个“新鲜/健康/便捷”关键词
- [ ] 已输出A/B/C三套切入点，且有明显差异
- [ ] 每套已标注适配场景与适配平台

## 第二步：根据切入点完成创意结构设计

### 目标
将选定切入点转化为可执行创意结构，明确“主题、风格、情绪节奏、转化动作”。
对应第一步的 A/B/C 三套切入点，输出 A/B/C 三套创意结构，并标注适配场景与平台。

### 平台适配建议表（默认策略）

当用户未指定平台策略时，优先按下表生成并微调：

| 平台 | 推荐切入方式 | 节奏建议 | 文案风格 | CTA建议 | 适配场景偏好 |
|------|--------------|----------|----------|---------|--------------|
| 抖音 | 冲突/反差/问题先抛出，前1-2秒直接建立张力 | 前3秒强吸引，中段快节奏递进，尾镜快速收口 | 短句、口语、强感叹，信息密度高 | 提问二选一/评论区互动/立即下单引导 | 通勤、加班、运动后、即时消费 |
| 小红书 | 生活方式切入，先给“可代入日常场景” | 节奏稍慢于抖音，强调细节质感与情绪连贯 | 真实分享口吻，避免过强叫卖感 | 收藏、留言经验、同款场景讨论 | 早餐、下午茶、野餐、居家轻食 |

平台适配补充规则（必须执行）：

- 同一套方案在不同平台复用时，至少调整以下三项：开场话术、字幕密度、CTA措辞
- 抖音版本优先强化“问题-答案”结构；小红书版本优先强化“场景-感受”结构；视频号版本优先强化“事实-信任”结构
- 若用户明确指定平台，以用户指定为最高优先级；建议表仅作为默认兜底

  - 视频号用 `B`（或 `C`）版本

### 创意结构设计方法

1. **确定核心主题**

2. **确定视频风格**
   - 情绪氛围向：唯美特写 + 生活化场景 + 柔和配乐
   - 生活化转化向：真实使用 + 口语台词 + 明确 CTA
   - 质感高级向：干净布景 + 少字强视觉 + 节奏克制

3. **设计内容结构（推荐3-5镜，10-12秒）**
   - 首镜 吸睛开场：情绪或冲突场景开场（可含人物），不露商品与品牌
   - 中段镜头 卖点递进：每镜头1个需求场景 + 1个感知卖点线索 + 1句口语台词 + 1句卖点文案（仍不露品牌）
   - 尾镜 汇聚收尾：商品完整露出 + 品牌露出 + 提问式金句 + CTA

4. **设置情绪节奏**
   - 0-3秒快速建立人物/场景氛围与需求张力
   - 3-9秒稳定输出卖点记忆点（通过情境推进，不做主图反复展示）
   - 9-12秒完成商品与品牌集中露出，并给出互动引导（如“你会选哪种心情”）

## 第三步：生成分镜脚本

### 目标
基于创意结构生成“可直接投喂AI工具”的分镜脚本，确保画面、台词、音效可执行。
要求前置镜头以情境叙事驱动，商品与品牌只在最后一镜集中露出；必要场合可出现人物与互动。
对应 A/B/C 三套创意结构，输出 A/B/C 三套完整分镜脚本，并标注适配场景与平台。

### 分镜结构

推荐 10-12 秒，3-5 镜头：

| 镜头 | 时长 | 内容 |
|------|------|------|
| 第1镜 | 0-2秒 | 需求情境开场（可有人物互动，不露商品品牌） |
| 第2镜 | 2-5秒 | 卖点线索A + 场景A（不露完整商品） |
| 第3镜 | 5-8秒 | 卖点线索B + 场景B（3镜方案可直接进入收尾） |
| 第4镜 | 8-10秒 | 卖点线索C + 场景C（4-5镜可选，仍不露品牌） |
| 第5镜 | 10-12秒 | 商品完整露出 + 品牌露出 + 金句收尾（必须保留） |

### 分镜脚本格式

每个镜头需包含：

```
镜头序号：
时长：
画面描述（含镜头运动/转场）：
场景信息（道具/环境/人物关系）：
台词/旁白（口语化，8-20字）：
镜头卖点文案（10-22字，强绑定用户需求）：
卖点字幕（主标+副标）：
情绪标签（如心动/清爽/治愈）：
背景音乐建议：
环境音效建议：
```
将上面字段中的“场景信息”按以下规则执行：前置镜头可写人物状态与轻互动，但需强调情境，不得提前露出品牌或完整商品主体。

### 画面限制规则（硬性执行）

- 前置镜头（第1-第4镜）禁止品牌露出：不得出现品牌名、logo、包装正面品牌信息
- 前置镜头（第1-第4镜）禁止完整商品主体长时间居中展示，避免“主图复读”
- 最后一镜（第5镜）必须完成商品完整露出与品牌露出，并承接 CTA
- 允许人物出现与互动（如拿取、分享、饮用、对话），但人物仅用于强化场景需求，不可成为主角
- 人物互动建议“轻量短时”：单镜头内人物动作描述不超过一句，避免喧宾夺主
- 若前置镜头误露品牌或完整商品主体，必须重写后再输出

### 镜头卖点文案生成规则（必须遵循）

- 每个镜头必须单独生成 1 条卖点文案，不可复用同一句
- 文案必须同时体现：
  - 1 个明确商品卖点（如原料、口感、新鲜度、工艺）
  - 1 个明确用户需求（如清爽、便捷、健康、治愈感）
- 结构建议：`卖点事实 + 用户收益`
- 字数建议 10-22 字，避免空泛形容词堆砌
- 禁止只写品牌口号，必须对应该镜头画面内容
- 镜头卖点优先使用“第0.5步高可信度卖点”，低可信度内容不得作为核心卖点

### 镜头卖点文案好/坏示例对照（给 qclaw）

**示例背景（芭乐樱樱茶）：**
- 镜头画面：晨光下带水珠的白芭乐茶饮置于长椅，瓶身轻微旋转
- 用户需求：清爽解腻、轻负担补水
- 商品卖点：四季香白芭乐，清甜不腻

**好示例（推荐）：**
- `四季香白芭乐清甜不腻，晨跑后这口更清爽。`
  - 命中卖点：四季香白芭乐、清甜不腻
  - 命中需求：晨跑后清爽补给
- `白芭乐果香轻盈不齁，运动后喝着刚刚好。`
  - 命中卖点：白芭乐果香、轻盈口感
  - 命中需求：运动后轻负担

**坏示例（避免）：**
- `春天就喝它，真的很好喝。`
  - 问题：无具体卖点、无用户需求、表达空泛
- `七鲜新品来了，快来买买买。`
  - 问题：只有促销口号，缺少镜头信息与价值说明
- `口感丰富层次多，品质非常高级。`
  - 问题：形容词堆砌，缺少可验证事实
- `这杯真好喝，快买。`
  - 问题：缺少具体卖点与场景需求，且过于硬广

**快速判定标准（逐镜检查）：**
- 是否能指出“具体卖点是什么”
- 是否能回答“满足了用户什么需求”
- 是否与镜头画面强关联（而不是放到任何镜头都成立）
